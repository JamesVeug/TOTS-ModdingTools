# Tales of the Shire Modding Tools

Welcome to the unofficial modding tools for Tales of the Shire!

This mod provides tools for you to 
- Extract files from the game
- Modify parts of the game using .json files 
- Add new things to the game.

Supports
- Adding/modifying Ingredients (Milk, Flour, Apples... etc)
- Adding/modifying Meals (Bread, Crispy Fish Pie... etc)
- Adding/modifying Recipes (Bread Recipe, Crispy Fish Pie Recipe... etc)
- Adding/modifying things for villagers to sell


## 1. How to install
1. Install v5.4.21 of [BepInEx](https://github.com/BepInEx/BepInEx/releases/tag/v5.4.21)
    - Extract the contents of the zip into `Steam\steamapps\common\TOTS` folder that has the games `TotS.exe` in it
    - You should now have
      - A folder called `TOTS/BepInEx`
      - A file called `TOTS/doorstop_config.ini`
2. Install the [Modding Tools mod](https://www.nexusmods.com/talesoftheshirealordoftheringsgame/mods/3)
    - Extract the contents of the zip into the `TOTS` folder
    - You should have
      - A folder called `TOTS/BepInEx/plugins/ModdingTools` with files in it
      - A folder called `TOTS/BepInEx/ExtraFiles` with ~8 files in it
   - Open `TOTS/doorstop_config.ini` with a text editor like NotePad
       - Change `dllSearchPathOverride=` to `dll_search_path_override = "ExtraFiles"`
3. Run the game
4. Confirm the settings menu has a Mods tab. If so then it installed correctly!

## 2. How to export information from the game
- Install BepInEx and Modding tools in #1
- Open the game
- Go to Settings
- Go to the Mods tab
- Turn on `Export Game to JSON`
- Load a save
- Wait until you can now walk around the world
- A new folder will be created that has all the .json files and images
  - `TOTS\BepInEx\plugins\TOTS_ModdingTools_Devs-ModdingTools\Exported`

## 4. How to add new items (Recipe, meal, ingredient... etc)

### 4.1 Ingredient (Items used for cooking and selling)
- Do the steps in #2 if you haven't yet
- Copy an existing Ingredient.json file from the Exported folder that is similar to what you want to add
  - eg: `Ingredient_Milk_itemType.json`
- Paste it somewhere in the plugins folder and rename it to what you want
  - e.g. `TOTS\BepInEx\plugins\Ingredient_Oil_itemType.json`
  - NOTE: Make sure it ends with `_itemType.json`
- Open it with any text editor (notepad, notepad++, VSCode... etc.)
- Change the `name` to anything representing your new item and be referenced by other items
    - eg: `Ingredient_Oil`
- Change the `displayName_en` to what you want it to be called in game
    - eg: `Oil`
- Change the `description_en` to what you want the description to be in the game
    - eg: `Yellow cooking oil made from olives. Yummy yummy!`
- Create a new image that is 128x128 and represents your item and put it in the plugins folder
  - eg: `TOTS\BepInEx\plugins\oil_item.png`
  - Change the `previewAspect.AutoGeneratedIcon` to the name of your image
    - eg: `oil_item.png`

> NOTE: After these steps the item should work in-game but there isn't any way to get it yet. 
> 
> Only way currently is to make it so an NPC can sell it. See steps #5

### 4.2 Meal (Items that can be eaten by villagers at shared meals)
- Do the steps in #2 if you haven't yet
- Copy an existing Meal.json file from the Exported folder that is similar to what you want
  - eg: `Meal_Bread_Onion_Focaccia_itemType.json`
- Paste it somewhere in the plugins folder and rename it to what you want
  - e.g. `TOTS\BepInEx\plugins\Meal_Lembas_Bread_itemType.json`
  - NOTE: Make sure it ends with `_itemType.json`
- Modify the meal as much as you want
  - Change the `name` to anything representing your new item and be referenced by other items
      - eg: `LembasBread`
  - Change the `displayName_en` to what you want it to be called in game
      - eg: `Lembas Bread`
  - Change the `description_en` to what you want the description to be in the game
      - eg: `Elvish waybread, light and sustaining.`
  - Change the `mealAspect.Recipe` to what recipe is required to create the meal. See #4.3 for how to add a recipe
      - eg: `Recipe_Lembas_Bread`
  - Change the `previewAspect.AutoGeneratedIcon` to the name of your image
      - eg: `lembas_bread.png`
      - Create a new image that is 128x128 and represents your item and put it in the plugins folder
        - eg: `TOTS\BepInEx\plugins\lembas_bread.png`

> NOTE: After these steps the Meal should work in-game but there isn't any way to get it yet.
>
> #4.3 to create a recipe to acquire the meal by cooking ingredients!

### 4.3 Recipe (Items that can be cooked to create meals)
- Do the steps in #2 if you haven't yet
- Copy an existing Recipe.json file from the Exported folder that is similar to what you want
  - eg: `Recipe_Bread_Onion_Focaccia_itemType.json`
- Paste it somewhere in the plugins folder and rename it to what you want
  - e.g. `TOTS\BepInEx\plugins\Recipe_Lembas_Bread_itemType.json`
  - NOTE: Make sure it ends with `_itemType.json`
- Modify the recipe as much as you want
  - Change the `name` to anything representing your new item and be referenced by other items
      - eg: `Recipe_Lembas_Bread`
  - Change the `displayName_en` to what you want it to be called in game
      - eg: `Lembas Bread Recipe`
  - Change the `description_en` to what you want the description to be in the game
      - eg: `Elvish waybread, light and sustaining.`
  - Change the `recipeAspect.FinishedProduct` to what meal this recipe creates. See #4.2 for how to add a meal
      - eg: `LembasBread`
  - Change the `recipeAspect.RecipeTrait` to what category the recipe belongs to
      - eg: `Bread`
  - Change the `recipeAspect.FixedIngredients` to what ingredients are 100% required to make the recipe. You can get the names by looking at the names in the .json files. 
      - eg: `Ingredient_Oil`, `Ingredient_Flour`, `Ingredient_Strawberry`, `Ingredient_Fennel`
  - Change the `recipeAspect.VariableTraits` to what ingredients with the given traits are required to make the recipe. You can get the traits by looking at the `cookingAspect.m_Traits` in ingredient .json files.
      - eg: `Bread`
  - Change the `previewAspect.AutoGeneratedIcon` to the name of your image
      - eg: `lembas_bread.png`
      - Create a new image that is 128x128 and represents your item and put it in the plugins folder
          - eg: `TOTS\BepInEx\plugins\lembas_bread.png`

> Note: After these steps the Recipe should work in-game and appear as a recipe when at the cooking station.

### 5.  Make a Villager sell your new item or existing item
- Do the steps in #2 if you haven't yet
- Copy an existing shopStock.json file from the Exported folder that is similar to what you want
  - eg: `Young_Tom_Cotton_Ingredient_Milk_shopStock.json`
- Paste it somewhere in the plugins folder and rename it to what you want
  - e.g. `TOTS\BepInEx\plugins\Young_Tom_Cotton_Ingredient_Oil_shopStock.json`
  - NOTE: Make sure it ends with `_shopStock.json`
- Modify the shopStock as much as you want
  - Change the `ItemName` to the name of the item you want sold
    - eg: `Ingredient_Oil`
  - Modify Items to indicate what relationship level they unlock and how much stock they have and how much they cost

> Note: After these steps the NPC should now sell your item in-game AFTER they have restocked and met your relationship level


### 6.  My thing didn't work. What do I do? HELP!
- Look at the BepInEx console for any red errors that show up. That should give clues to what went wrong
  - Or load the log file in a text editor and look for errors: `TOTS\BepInEx\LogOutput.log`
- Common problems are
  - There's a space in the `name` of the item you added so it doesn't match
  - Case of `name` does not match 
    - eg: `Ingredient_oil` vs `Ingredient_Oil` will NOT work
  - Dimensions of images are not correct or your image is missing or incorrectly names in the .json
